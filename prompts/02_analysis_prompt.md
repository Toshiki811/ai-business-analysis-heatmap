# 02 業務分析・スコアリングプロンプト

## 目的

`input/normalized/` 内の整形済みテキストを読み込み、AI導入効果を評価した
構造化JSONファイル（`output/analysis_YYYYMMDD.json`）を生成する。

## 入力

- `input/normalized/` 内の全 .md / .txt ファイル

## 出力

- `output/analysis_YYYYMMDD.json`（YYYYMMDDは実行日付）

---

## 分析手順

### Step 1: 業務フロー（横軸）の抽出

ドキュメント全体を読み、業務の流れを2段階で整理する。

**大分類フロー（group）**：業務全体を5〜8つの大きなフェーズに分ける
- 例：「受付」「情報収集」「判断・回答」「後処理」「報告」

**具体ステップ（steps）**：各大分類の中に含まれる個別の作業を列挙する
- 1つの大分類に2〜4ステップが目安
- ステップ名は10文字以内のコンパクトな表現にする

### Step 2: 業務カテゴリー（縦軸）の抽出

ドキュメントに登場する業務の種類・性質を軸にカテゴリーを5〜8個定義する。
- 例：「顧客対応」「情報検索」「判断・確認」「文書作成」「記録・報告」「承認フロー」
- 各ドキュメントの内容から実態に合ったカテゴリーを設定すること

### Step 3: 各セルのスコアリング

業務フローの各ステップ × 各業務カテゴリーの組み合わせで該当するものを評価する。

**該当判定**：そのステップでそのカテゴリーの業務が発生するかどうかを判断する。
発生しない組み合わせはスキップしてよい（セル数が多すぎる場合は主要なもののみ）。

**スコア計算式**：
```
AI導入効果スコア（100点満点）=
  作業時間削減インパクト × 0.45
+ 作業頻度・件数       × 0.25
+ 実装容易性           × 0.20
+ 品質改善・ミス削減効果 × 0.10
```
※ 各項目は5段階評価 → 合計を100点に換算（最大5.0 → 100点）

**各評価軸の基準**：

| 評価軸 | 5 | 4 | 3 | 2 | 1 |
|---|---|---|---|---|---|
| 作業時間削減インパクト | 1件30分以上削減 | 15〜30分 | 5〜15分 | 1〜5分 | 1分未満 |
| 作業頻度・件数 | 毎日50件以上 | 毎日10〜50件 | 週数回/数件 | 月数回 | ほぼ発生しない |
| 実装容易性 | 既存APIで即実装可 | 軽微なカスタマイズで対応 | 中程度の開発が必要 | 大規模開発 | 技術的に困難 |
| 品質改善・ミス削減効果 | ミスが頻発・重大な影響 | 時々ミスがある | 品質のばらつきがある | ほぼ品質は安定 | 品質改善不要 |

**開発規模の判定**：
- 実装容易性スコア 5 → 開発規模「小」
- 実装容易性スコア 3〜4 → 開発規模「中」
- 実装容易性スコア 1〜2 → 開発規模「大」

### Step 4: TOP3の選定とTo-Be業務フロー作成

総合スコア上位3件を TOP3 として選定し、それぞれについて以下を作成する。

**As-Is フロー**：現状の業務ステップ（全て Human）

**To-Be フロー**：AI導入後の業務ステップ。以下の actor を使い分ける。
- `Human`：人間が実施するステップ
- `AI`：AIが自動処理するステップ
- `Human Review`：AIの処理結果を人間が確認・承認するステップ
- `System`：既存システムが自動処理するステップ

## 出力JSONの厳密なスキーマ

```json
{
  "metadata": {
    "title": "業務プロセスAI導入分析",
    "created_at": "YYYY-MM-DD",
    "source_documents": ["ファイル名1", "ファイル名2"]
  },
  "flow_columns": [
    {
      "group": "大分類フロー名",
      "steps": ["ステップ1", "ステップ2", "ステップ3"]
    }
  ],
  "categories": ["カテゴリー1", "カテゴリー2"],
  "heatmap_cells": [
    {
      "category": "業務カテゴリー名",
      "flow_step": "ステップ名（flow_columnsのstepsに含まれる値と一致させること）",
      "score": 87,
      "time_reduction_score": 5,
      "frequency_score": 4,
      "implementation_ease_score": 4,
      "quality_impact_score": 3,
      "estimated_time_saved": "1件あたり5〜10分",
      "development_scale": "小",
      "ai_use_case": "AIで何をするか1〜2文で説明",
      "reason": "なぜこのスコアになるかの根拠",
      "source_reference": "ドキュメントの該当箇所の概要",
      "is_top3": true
    }
  ],
  "top3": [
    {
      "rank": 1,
      "title": "施策タイトル（20文字以内）",
      "target_category": "対象の業務カテゴリー",
      "target_flow_step": "対象のフローステップ",
      "current_issue": "現状の課題（1〜2文）",
      "ai_solution": "AI導入の具体的な解決策（1〜2文）",
      "expected_effect": "期待される効果（箇条書き可）",
      "risks": "リスク・注意点（1〜2文）",
      "as_is_flow": [
        {
          "step": "ステップ名",
          "actor": "Human",
          "description": "このステップで何をするか"
        }
      ],
      "to_be_flow": [
        {
          "step": "ステップ名",
          "actor": "Human | AI | Human Review | System",
          "description": "このステップで何をするか"
        }
      ]
    }
  ]
}
```

## 注意事項

- `flow_step` の値は必ず `flow_columns[].steps[]` に含まれる文字列と完全一致させること
- スコアは根拠のある評価にすること（推測でも理由を `reason` に明記）
- TOP3は `heatmap_cells` の `is_top3: true` と `top3` 配列の両方に含めること
- JSONは valid なフォーマットで出力すること（コメント不可）
