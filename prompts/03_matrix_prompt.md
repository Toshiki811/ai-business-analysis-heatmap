# 03 業務整理マトリクス生成プロンプト

## 目的

`input/normalized/` の正規化済みドキュメントをもとに、業務整理マトリクスを生成する。

この工程ではクライアント入力CSV、スコアリング、draw.io、HTML生成は行わない。

## 入力

- `input/normalized/` 内の全 `.md` / `.txt` ファイル
- `output/flow_axes_YYYYMMDD.json`（存在する場合は業務分類、分類別横軸、ヒートマップ抽象軸の正本として使う）
- `input/source/client_input_filled.csv` または `input/source/client_input_filled_YYYYMMDD.csv`（存在する場合。回答済み設問の再掲防止に使う）

## 出力

`output/matrix_YYYYMMDD.md`

```markdown
| 業務分類 | 業務種別 | タスク順 | マトリクス横軸 | ヒートマップグループ | ヒートマップステップ | タスク名 | 業務内容詳細 | 1件あたり所要時間_分 | 人手の負担 | 現状課題 | AI導入余地 | クライアントに確認したいこと | 区分 |
```

## 列定義

| 列名 | 定義 |
|---|---|
| 業務分類 | 大括りの業務グループ。1ドキュメントで5〜8分類を目安に抽出 |
| 業務種別 | 業務分類の下位区分 |
| タスク順 | As-Isフロー用の実作業順。業務種別ごとに1から始まる連番 |
| マトリクス横軸 | 業務分類内のタスク粒度の横軸名。必ず該当業務分類の `category_flow_columns[業務分類].steps[]` のいずれかと完全一致 |
| ヒートマップグループ | ヒートマップ2段横軸の上段。必ず `heatmap_columns[].group` のいずれかと完全一致 |
| ヒートマップステップ | ヒートマップ2段横軸の下段。必ず該当グループの `heatmap_columns[].steps[]` のいずれかと完全一致 |
| タスク名 | タスクの簡潔な名称（10〜20字） |
| 業務内容詳細 | 担当者・使用システム・判断基準を含む具体作業 |
| 1件あたり所要時間_分 | 文書記載値を優先。不明なら `5〜10（仮）` のように仮置き |
| 人手の負担 | `低` / `中` / `高`。不明なら仮置き |
| 現状課題 | 文書記載の課題。なければ `要確認` |
| AI導入余地 | `低` / `中` / `高`。繰り返し性・ルール明確性で仮評価 |
| クライアントに確認したいこと | As-Isフロー改善に直結する不明点のみ。質問文内に回答例を併記する |
| 区分 | 初回は必ず `仮案`。回答済み行は `ヒアリング済` |

## 確認事項の作成ルール

- 担当者、承認者、処理順序、分岐条件、例外処理、差戻し、再承認、システム連携など、回答によりAs-Isフローが変わる質問だけを書く。
- 資料形式、件数、頻度、所要時間、AI導入余地、単なる出力可否、システム活用状況など、As-Isフローの配置・順序・分岐を変えない情報は書かない。定量情報は `04_client_csv_prompt.md` のCSV列で収集する。
- 入力文書から担当者、承認者、使用システム、処理順序が十分に読み取れる場合は確認事項を書かない。
- 入力文書の「必要に応じて確認」「担当・承認・例外処理は必要に応じて確認」のような一般的な注記だけを根拠に、全タスクへ確認事項を作らない。
- 保管場所、管理媒体、フォーマット、電子化状況、単なる出力可否など、フロー図のノード・レーン・矢印・分岐を変えない運用詳細は確認事項にしない。
- 質問文には必ず回答例を併記し、形式は `質問: ... / 回答例: ...` に統一する。
- 回答例はクライアントがそのまま選ぶ選択肢ではなく、回答粒度をそろえるための記入例にする。
- 回答済み、または `（確認不要）` の設問は再掲しない。
- 該当する疑義がなければ空欄でよい。

## ルール

- 業務種別ごとにタスク順を振り直す。
- `output/flow_axes_YYYYMMDD.json` が存在する場合、`マトリクス横軸` は必ず該当業務分類の `category_flow_columns` から選ぶ。存在しない横軸名を新規に作らない。
- `ヒートマップグループ` と `ヒートマップステップ` は必ず `heatmap_columns` から選ぶ。具体タスクを比較可能な抽象ステップへ対応付ける。
- `output/flow_axes_YYYYMMDD.json` が存在しない場合は、先に `02_flow_axes_prompt.md` を実行してから本工程を実行することを推奨する。
- 同じ `業務分類 × 業務種別 × マトリクス横軸` に複数タスクが該当する場合は、タスクを分けて行を作成してよい。
- 旧成果物を再利用する場合のみ、旧 `共通ステップ` は `マトリクス横軸` および `ヒートマップステップ` のフォールバックとして読み替えてよい。
- 数値が不明な場合は仮置きであることを明示する。
- 担当者や使用システムが明示されていない場合でも、フローの配置・順序・分岐に影響しないなら確認事項を空欄にする。
- セルが長くなる場合は80字を目安に短縮してよい。
- マトリクスは初回レビュー用の仮案として、過度に確定的に書かない。
