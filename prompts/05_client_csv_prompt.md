# 05 クライアント入力CSV生成プロンプト

## 目的

`output/matrix_YYYYMMDD.md` と詳細As-Isフローの確認事項をもとに、クライアントがヒアリング結果を入力できるCSVを生成する。

この工程ではマトリクスの再作成、スコアリング、draw.io、HTML生成は行わない。

## 入力

- `output/matrix_YYYYMMDD.md`
- `output/asis_flows_YYYYMMDD.json`（存在する場合。`hearing_items` を確認事項として取り込む）
- `input/source/client_input_filled.csv` または `input/source/client_input_filled_YYYYMMDD.csv`（存在する場合。回答済み設問の再掲防止に使う）

## 出力

`output/client_input_YYYYMMDD.csv`

## CSV列定義

| 列名 | 内容 |
|---|---|
| 業務分類 | マトリクスの値をコピー |
| 業務種別 | マトリクスの値をコピー |
| タスク順 | マトリクスの値をコピー |
| マトリクス横軸 | マトリクスの値をコピー。業務分類別の横軸名 |
| ヒートマップグループ | マトリクスの値をコピー。ヒートマップ2段横軸の上段 |
| ヒートマップステップ | マトリクスの値をコピー。ヒートマップ2段横軸の下段 |
| タスク名 | マトリクスの値をコピー |
| 業務内容詳細 | マトリクスの値をコピー |
| 1件あたり所要時間_分_仮案 | マトリクスの `1件あたり所要時間_分` |
| 1件あたり所要時間_分_ヒアリング後 | 空欄 |
| 人手の負担_仮案 | マトリクスの `人手の負担` |
| 人手の負担_ヒアリング後 | 空欄 |
| 月間件数 | 空欄 |
| 現状課題 | マトリクスの値をコピー |
| 確認したいこと | マトリクスの `クライアントに確認したいこと` に加え、`output/asis_flows_YYYYMMDD.json` の `hearing_items[]`（分岐・差戻し・例外の確認質問）を該当タスクの行へ転記。`質問: ... / 回答例: ...` 形式をそのまま出力し、同一内容の質問は1つに統合、回答済み・確認不要は空欄 |
| クライアント回答 | 空欄 |
| AI導入余地_仮案 | マトリクスの `AI導入余地` |
| As-Isフロー更新内容_業務分類 | 空欄。業務分類単位で、As-Isフローへ反映したい担当者、順序、分岐、例外、差戻し、システム連携などを自由記入 |
| 備考 | 空欄 |

## 出力ルール

- 文字コードはUTF-8 with BOMにする。
- 区切り文字はカンマ。
- セル内のダブルクォートは `""` にエスケープする。
- セル内にカンマ、ダブルクォート、改行が含まれる場合はダブルクォートで囲む。
- セル内改行はスペースに置換する。
- ヘッダー行を1行目に出力する。
- `output/matrix_YYYYMMDD.md` が `## 業務分類: ...` / `### 業務種別: ...` の階層形式の場合は、直近の見出し値を各タスク行の `業務分類` / `業務種別` としてCSVへ出力する。
- `マトリクス横軸`、`ヒートマップグループ`、`ヒートマップステップ` は `output/matrix_YYYYMMDD.md` の値をそのままコピーし、名称変更しない。
- 旧形式のマトリクスに `共通ステップ` しかない場合は、`共通ステップ` を `マトリクス横軸` と `ヒートマップステップ` に読み替え、`ヒートマップグループ` は `flow_axes_YYYYMMDD.json` の対応グループから補完する。
- `確認したいこと` は質問文と回答例を分割せず、既存列にそのまま出力する。回答例用の列は追加しない。
- `確認したいこと` はAs-Is/To-Beフローのノード、レーン、順序、分岐、例外処理、システム連携を変える未確定情報だけを出力する。
- `As-Isフロー更新内容_業務分類` はクライアント回答とは分け、確認事項単位ではなく業務分類単位の補足・修正内容を記入する列として、確認事項・回答関連列の最後（`備考` の直前）に出力する。
- 旧CSVの `As-Isフロー更新内容` は後続工程で互換入力として読み込んでよいが、新規CSVには出力しない。
- 入力文書から担当者、承認者、使用システム、処理順序が十分に読み取れる行や、資料形式・件数・頻度・所要時間・単なる出力可否・保管場所・管理媒体・フォーマットだけを聞く行は空欄にする。

## ヒアリング後ファイルの扱い

- クライアント入力後のファイルは `input/source/client_input_filled.csv` または `input/source/client_input_filled_YYYYMMDD.csv` として保存する。
- 複数存在する場合、後続分析では更新日時が最新の1件を使用する。
- `クライアント回答` が入力済み、`（確認不要）`、または入力文書から推定可能な行は次回CSVで `確認したいこと` と `クライアント回答` を空欄にする。
